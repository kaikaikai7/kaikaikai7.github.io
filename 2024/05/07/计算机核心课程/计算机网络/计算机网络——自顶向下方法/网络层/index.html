网络层

1. 简介

1.1 网络层

  • 发送方：将报文段封装为数据报
  • 接收方：将报文段传送到传输层
  • 每个主机、路由器中都有网络层协议
  • 路由器检查通过它的所有 IP 数据报的头部字段

1.2 路由和转发

  • forwarding （转发）：当一个分组到达某路由器的一条输入链路时，该路由器必须将该分组移动到适合的输出链路。
  • routing（路由）：确定分组从源到目的地的路由或路径。
  • 每台路由器都有一个转发表，通过检查到达分组首部的一个或多个字段值进行转发，使用这些首部值在转发表中索引。路由算法决定了路由器转发表中的值。

1.3 连接建立

  • 某些网络架构中的第三种重要的网络层功能：
    • 连接建立（连接建立）
    • ATM、帧中继、X.25、MPLS（非因特网）
  • 在数据报流动之前，两台终端主机和中间路由器建立虚拟连接。（注：TCP只需要通信双方）
  • 网络层与传输层连接服务：
    • 网络层：两个主机之间
    • 传输层：两个进程之间

1.4 网络层服务模型

  • 单个数据报的服务示例：
    • 确保交付
    • 具有时延上界
  • 数据报流服务示例：
    • 有序分组交付
    • 确保最小带宽
    • 确保最大时延抖动
    • 安全性
  • 因特网的网络层仅提供尽力而为服务（无服务）

2. 虚电路和数据报网络

2.1 网络层有连接和无连接服务

分组交换网络的类型：

  • 数据报网络（如因联网）：提供网络层无连接服务
  • VC 网络（如 ATM、帧中继、X.25、MPLS）：提供网络层有连接服务

传输层服务和网络层服务：

传输层 网络层
服务 进程到进程 主机到主机
选择 两者都有 无连接或有连接
实现 终端系统 网络核心和终端系统

2.2 虚电路网络

  • VC（virtual circuit）组成：
    • 从源到目的地的路径（链路和路由器）
    • VC号码：路径上每条链路都有一个编号
    • 路径上路由器转发表中的条目
  • VC中每个分组都有VC号码，而不是IP地址。
  • VC 号码在每条链路上可能不同，新 VC 号码来自转发表。

2.2.1 转发表

路由器维护连接状态信息！

2.2.2 VC建立

  • 发送方传输层与网络层联系，指定接收方地址，等待网络建立 VC；
  • 网络层确定发送方和接收方之间的路径，即 VC 的所有数据包将经过的一系列链路和路由器。 指定源和目的地之间的所有数据包都将走相同的路径；
  • 网络层确定路径上每个链路的 VC 号码；
  • 网络层在每个路由器的转发表中添加一个条目。

2.2.3 信令协议

  • signaling message （信令报文）：端系统向网络发送指示虚电路启动与终止的报文，以及路由器间传递的用于建立虚电路的报文。
  • signaling protocols（信令协议）：用于交换这些报文的协议。
  • 用于设置、维护和终止 VC ；用于 ATM、帧中继、X.25 未用于当今因特网。

2.2.4 区别

网络层的 VC 建立与传输层的连接建立之间的区别：

  • 传输层的连接建立只涉及两终端系统
    • 两终端系统单独决定传输层连接的参数
    • 两终端系统知道传输层连接，路由器完全不知道
  • VC 网络层，两终端系统之间路径上的路由器参与 VC 建立
    • 每个路由器完全知道经过它的所有 VC

2.3 数据报网络

  • 无连接建立，路由器不维护任何状态信息；
  • 分组使用目标主机地址，从而进行转发；
  • 同一源——目的地址对之间的数据包可能会走不同的路径。

3. 路由器的内部结构

路由器的两个关键功能：

  • 运行路由算法/协议（RIP、OSPF、BGP）
  • 将数据报从传入链路转发到传出链路

组成部分：

  • 输入端口
  • 交换结构
  • 输出端口
  • 路由选择处理器

4. IP：Internet Protocol

4.1 数据报格式

  • 版本 : 有 4（IPv4）和 6（IPv6）两个值；

  • 首部长度 : 占 4 位，因此最大值为 15。值为 1 表示的是 1 个 32 位字的长度，也就是 4 字节。因为固定部分长度为 20 字节，因此该值最小为 5。如果可选字段的长度不是 4 字节的整数倍，就用尾部的填充部分来填充。

  • 区分服务 : 用来获得更好的服务，一般情况下不使用。

  • 总长度 : 包括首部长度和数据部分长度。

  • 标识 : 在数据报长度过长从而发生分片的情况下，相同数据报的不同分片具有相同的标识符。

  • 片偏移 : 和标识符一起，用于发生分片的情况。片偏移的单位为 8 字节。

  • 生存时间 ：TTL，它的存在是为了防止无法交付的数据报在互联网中不断兜圈子。以路由器跳数为单位，当 TTL 为 0 时就丢弃数据报。

  • 协议 ：指出携带的数据应该上交给哪个协议进行处理，例如 ICMP、TCP、UDP 等。

  • 首部检验和 ：数据报每经过一个路由器，都要重新计算检验和，因为TTL 字段值会变，选项字段也可能会变。因此检验和不包含数据部分可以减少计算的工作量。

Q：为什么 TCP/IP 在传输层和网络层都进行错误检查？

A：IP层的校验和只针对IP头部进行校验，Segment数据在传输层进行校验。IP和TCP/UDP可能属于不同的协议栈。

4.1.1 比较：长度字段

  • UDP：
    • 头部字段固定为8个字节
    • 长度字段（16 位）：UDP 报文段的字节数（头部+数据）
  • TCP：
    • 头部字段典型长度为20个字节 （当option字段为空时）
    • 头部长度字段（4 位）：包括选项字段在内的 TCP 报头长度，以 32 位字衡量（以 4 个字节为基本单位）
  • IP（IPv4）：
    • 头部字段典型长度为20个字节（当option字段为空时）
    • 首部长度字段（4 位）：包括选项字段在内的头部长度（以 4 个字节为基本单位）
    • 总长度字段（16 位）：IP 数据报的总长度，以字节为单位（头部+数据）

4.1.2 IP分片重组

  • MTU (max transfer unit，最大传输单元)：链路层帧能携带的最大数据报长度。
    • 不同的链路类型，不同的 MTU
    • 以太网帧 MTU=1500 字节，某些广域链路帧的传输量不超过 576 字节
  • MSS（maximum segment size，最大报文段长度)：传输层能承载的应用层最大数据长度，不包括传输层的头部。
  • 例如：一个在以太网上传输的IPv4 TCP报文的MSS值为1460，是由MTU=1500bytes减去IPv4 Header（20 Byte）和TCP header（20 Byte）得到。
  • 问题：路由上的每条链路可以使用不同的链路层协议，可以有不同的 MTU 。
  • 大型 IP 数据报在网络内被分片：
    • 一个数据报变成多个数据报
    • 仅在目的地重组
    • IP 头比特用于识别相关分片并对其进行排序

4.2 IPv4寻址

4.2.1 简介

  • IP 地址：主机、路由器接口的 32 位标识符，2^32 个可能的 IP 地址 (40 亿)。
  • 接口：主机/路由器与物理链路之间的边界。
    • 路由器通常有多个接口
    • 主机通常有一个或两个接口（如有线以太网、无线802.11）
  • IP地址和一个接口相关联，而不是与包括该接口的主机或路由器相关联。

4.2.2 子网

  • IP 地址子网部分相同的设备接口，可以在没有中间路由器参与的情况下物理地相互连接。
  • 要确定子网，将每个接口与其主机或路由器分离，创建孤立的网络孤岛。每一个网络孤岛叫作子网。
  • 子网：223.1.1.0/24
    • /24 表示子网掩码：11111111 11111111 11111111 00000000 = 255.255.255.255.0
    • 子网网络地址：223.1.1.0（计算方式：IP地址&子网掩码）

4.2.3 计算

已知一个IP地址和子网掩码可以算出：

  • 子网的网络地址(network address)；
  • 子网的广播地址(broadcast address);
  • 子网中的主机地址范围；
  • 子网中有几台主机。

4.2.4 IP地址分配

4.2.4.1 CIDR

  • CIDR：Classless InterDomain Routing（无类别域间路由选择）
  • 地址格式：a.b.c.d/x，其中 x 是地址中子网部分的 # 位，称为前缀
  • 组织内设备的 IP 地址将共享共同的前缀，路由器只考虑前缀来转发
  • 地址的其余 32-x 位用于区分组织内的设备

4.2.4.2 分类编址

  • classful addressing（分类编址）：IP 地址的网络部分长度限制为 8、16 或 24 比特
  • 具有 8 位、16 位和 24 位子网地址的子网被称为 A 类、B 类和 C 类网络
  • 快速消耗，利用率低

4.2.4.3 私有IP地址

  • 随着私有IP网络的发展，为节省可分配的注册IP地址，有一组IP地址被拿出来专门用于私有IP网络，称为私有IP地址。 这些地址不会被Internet分配，在Internet上也不会被路由，虽然它们不能直接和Internet连接，但通过技术手段仍旧可以和 Internet通信（NAT技术）
    • A类私有IP地址空间: 10.0.0.0~10.255.255.255，即10.0.0.0/8
    • B类私有IP地址空间:172.16.0.0~172.31.255.255，即172.16.0.0/12
    • C类私有IP地址空间:192.168.0.0~192.168.255.255，即192.168.0.0/16
  • 一些特殊的IP地址
    • 0.0.0.0 ：保留地址，表示本网络上的本主机（设备启动时但又不知道自己的IP地址情况下）
    • 255.255.255.255：受限广播地址，只在本网络上进行广播（各个路由器均不转发）。（IP地址的主机字段全为1，例如：192.168.10.255，称为直接广播地址，被发送到该网络号的每台主机，路由器可转发）
    • 127.0.0.1：环回地址(Loopback Address)，一般用来作为本地软件环回测试（lookback test）本主机的进程之间通信。

4.2.5 分层寻址

地址聚合/路由聚合：使用单个网络前缀（200.23.16.0/20）通告多个网络（子网）的能力。

4.4 IPv6

4.4.1 数据报格式

  • 固定40字节，IP地址长度为128位。
  • 不进行分片重组，只能在源和目的地上使用。
  • 流量类别（优先级）：确定数据流中数据报的优先级。
  • 流量标签：确定同一 "流 "中的数据报（"流 "的概念没有明确定义）。
  • 下一报头：确定数据的上层协议。

4.4.2 过渡方法

从 IPv4 过渡到 IPv6 的两种方法：双栈（dual-stack）和隧道（tunneling）。

4.4.2.1 双栈

  • IPv6 节点具有完整的 IPv4 实施功能
    • 被称为 IPv6/IPv4 节点
    • 同时发送和接收 IPv4 和 IPv6 数据报
    • 同时拥有 IPv4 和 IPv6 地址
  • 如果发送方或接收方仅支持 IPv4，则必须使用 IPv4 数据报。

4.4.2.2 隧道

  • IPv6 数据报作为 IPv4 数据报的有效载荷在 IPv4 路由器之间传输

5. 路由算法

路由算法：找到从源路由器到目的路由器的最小代价路径的算法。

分类一：

  • 全局式路由算法
    • 所有路由器都有完整的拓扑结构、链路成本信息
    • 称为链路状态（Link State）算法
  • 分散式路由算法
    • 路由器只知道物理连接的邻居，与邻居的链接成本
    • 迭代计算过程，与邻居交换信息
    • 称为距离向量（Distance Vector）算法

分类二：

  • 静态路由算法
    • 路由随时间缓慢变化
    • 人工手动编辑转发表
  • 动态路由算法
    • 路由变化更快
    • 定期更新，或响应拓扑或链路成本的变化

分类三：

  • 负载敏感路由算法
    • 链路开销动态变化，以反映当前的拥塞程度
    • 高成本与当前拥塞的链路有关
    • 路由算法将倾向于选择绕过拥塞链路的路由
    • 如：ARPAnet
  • 负载不敏感路由算法
    • 链路开销没有明确反映当前（或最近过去）的拥塞水平
    • 如RIP、OSPF、 BGP

5.1 链路状态算法

  • 在 LS 路由算法中，网络拓扑和所有链路成本都是已知的
  • 实现途径：通过让每个节点向网络中所有其他节点广播链路状态数据包来实现链路状态广播算法（容易形成广播风暴）
    • 每个链路状态数据包都包含其附属链路的身份和成本
    • 最后，所有节点都有一个完全相同的网络视图
  • 链路状态路由算法被称为迪杰斯特拉算法（Dijkstra's algorithm）
  • 迭代算法：经过 k 次迭代后，知道到 k 个目的地的最小成本路径

5.1.1 数学符号

  • c(x,y)：从节点x到y的链路代价，如果不相邻则值为无穷
  • D(v)：从源点到目标节点v的最小代价路径的值
  • p(v)：从源点到目标节点v的最小代价路径中，节点v的前继节点
  • N'：已经确定在最小代价路径中的节点集合

5.3 层次化路由

  • 背景：规模、管理自治

  • 解决方案：将路由器聚合成区域，即自治系统（autonomous systems，AS）

  • 同一 AS 中的路由器运行同一路由协议

    • 自治系统域内路由协议（“intra-AS” routing protocol）
    • 不同 AS 中的路由器可以运行不同的域内路由协议

  • 网关路由器

    • 在自己 AS 的边缘
    • 与另一个 AS 中的路由器有链接

  • 一个路由器如何将一个数据包路由到AS之外？域间路由协议（inter-AS routing protocol）

    • 学习可达性信息
    • 将此可达性信息传播给该域内所有路由器

  • 因特网中的所有 AS 都使用相同的域间路由协议（BGP4）

  • 转发表由域内和域间路由算法共同配置

    • 域内路由算法为内部目的地设置条目
    • 域内、域间路由算法共同为外部目的地设置条目

  • 热土豆路由（hot potato routing）：源路由器距离网关路由器最短路径代价中最小的网关，作为其转发表对应的接口。

6. 因特网中的路由选择

  • 自治系统域内路由（Intra-AS Routing）
    • AS 内路由选择用于确定 AS 内路由选择的执行方式
    • 又称内部网关协议（interior gateway protocols，IGP）
    • 最常见的域内路由协议（intra-AS routing protocols）
      • OSPF：Open Shortest Path First 开放最短路径优先
      • RIP：Routing Information Protocol 路由信息协议
      • IGRP：Interior Gateway Routing Protocol（Cisco proprietary）
  • 自治系统域间路由（Inter-AS Routing）
    • BGP：Border Gateway Protocol 边界网关协议

6.1 OSPF

  • OSPF 部署在上层互联网服务提供商中，而 RIP 则部署在下层互联网服务提供商和企业网络中

  • 使用链路状态算法

    • 路由器向自治系统内所有其他路由器广播路由选择信息
    • 每个节点都有拓扑图
    • 使用 Dijkstra 算法进行路由计算

  • OSPF 广播为每个邻居提供一个条目

  • OSPF 报文通过IP数据报承载（注：RIP由UDP报文承载）

  • 优点：

    • 安全性：所有 OSPF 报文都经过验证

    • 允许多条相同开销的路径（RIP 中只允许一条）

    • 集成单播和多播支持

    • 支持在单个AS中的层次结构

      • 两级层次结构：局域网、主干网
      • 区域边界路由器：汇总自己区域内网络的距离，向其他区域边界路由器发布
      • 骨干路由器：运行仅限于骨干的 OSPF 路由
      • 边界路由器：连接其他 AS

6.2 RIP

  • 使用距离向量算法
    • 距离指标：hops（最大为15），每条链路代价为1
    • 每隔30秒在RIP响应消息中与邻居交换距离向量
    • 每条广播：最多25个目标子网的列表
  • 注：距离向量
    • 先前部分：开销在路由器对之间定义
    • RIP：源路由器到目标子网，包括目标子网
  • 链路故障与恢复
    • 如果180秒没有收到广播，则宣布邻居/链路死亡
      • A 通过邻居的路由已失效
      • 向 A 的其他邻居发送新广播
      • 其他邻居依次发送新广播（如果表格发生变化）
    • 毒性逆转：距离 =16 hops
  • RIP 路由表
    • 由名为 route-d（守护进程）的应用级进程管理
    • 广播以 UDP 数据包形式发送，使用端口号 520，定期重复发送

6.3 BGP

  • BGP 为每个 AS 提供
    • eBGP：从相邻 AS 获取子网可达性信息
    • iBGP：向所有 AS 内部路由器传播可达性信息
    • 根据可达性信息和策略，确定通往其他网络的良好路由
  • 允许子网向因特网其他部分宣布自己的存在

6.3.1 基本知识

  • BGP 会话：两个 BGP 路由器（"对等方"）交换 BGP 消息
    • 通告通向不同目标网络前缀的路径（"路径向量 "协议）
    • 通过使用端口 179 的半永久 TCP 连接进行交换
  • 目的地格式：目的地不是主机，而是 CIDR 化的前缀
  • 路由器通过BGP连接通告前缀时，它还包含一些BGP属性，带有属性的前缀即一条路由
  • 两个重要属性
    • AS-PATH：包含了通告已通过的 AS列表；用于检测和防止循环通告。假定前缀138.16.64/24 首先是由AS2向ASl通告的；如果ASl接下来将该前缀向AS3通告，则该AS PATH 将是AS2 ASl。
    • NEXT-HOP：表示下一跳 AS 的特定内部AS 路由器接口

6.3.2 路由选择

如果对相同前缀存在2条及以上路由，BGP顺序调用下列消除规则：

  • 本地偏好值属性
  • 最短AS-PATH
  • 最靠近NEXT-HOP路由器
  • 其他BGP标准
