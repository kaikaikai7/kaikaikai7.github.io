哈希表篇

1. 哈希表的查找、插入及删除

设计哈希集合

实现了一个基于链表的哈希集合数据结构，并提供了添加、删除、查询和释放内存的功能。注：链表中使用了虚拟头节点。

设计哈希映射

「设计哈希映射」与「设计哈希集合」解法接近，唯一的区别在于我们存储的不是 key 本身，而是 (key,value) 对。除此之外，代码基本是类似的。

存在重复元素

法一：在对数字从小到大排序之后，数组的重复元素一定出现在相邻位置中。因此，我们可以扫描已排序的数组，每次判断相邻的两个元素是否相等，如果相等则说明存在重复的元素。注：冒泡排序会超时。

qsort 是 C 标准库中的一个函数，用于对数组进行快速排序，时间复杂度为 O(n*log(n))。它的声明如下：

  • base -- 指向要排序的数组的第一个元素的指针。
  • nitems -- 由 base 指向的数组中元素的个数。
  • size -- 数组中每个元素的大小，以字节为单位。
  • compar -- 用来比较两个元素的函数。

法二：对于数组中每个元素，我们将它插入到哈希表中。如果插入一个元素时发现该元素已经存在于哈希表中，则说明存在重复的元素。

两个数组的交集

遍历数组 nums1，对于其中的每个元素，遍历数组 nums2 判断该元素是否在数组 nums2 中。使用了哈希表降低时间复杂度，以空间换时间。

最长连续序列

法一：通过排序数组，并使用线性扫描的方式，以时间复杂度 O(n log n) 找到最长连续序列的长度。

法二：

  1. 首先，将数组中的所有元素存入一个哈希表中，以便快速检查某个元素是否存在。
  2. 然后，对数组中的每个元素进行遍历，对于每个元素，向两个方向（正方向和负方向）搜索连续序列，直到找到序列的结束。在此过程中，记录当前连续序列的长度。
  3. 遍历数组中的每个元素并执行上述操作，最终得到数字连续的最长序列的长度。

法三：在法二基础上进行修改，遍历元素时，先检查该元素是否是连续序列起电，如果是，则进入循环；否则不进行操作。

单词规律

  1. 初始化哈希表： 创建两个哈希表，一个用于存储字符到字符串的映射关系，另一个用于存储字符串到字符的映射关系。因为哈希表是通过键进行查询的，因此需要两个。
  2. 遍历模式字符串： 遍历模式字符串的每个字符，并同时提取目标字符串中的单词。
  3. 检查映射关系： 对于当前字符和单词，检查两个哈希表中的映射关系是否一致，若不一致则返回 false。
  4. 更新映射关系： 若不存在映射关系，则更新两个哈希表中的映射关系。
  5. 处理下一个单词： 继续处理下一个单词，直到模式字符串遍历完毕或者目标字符串遍历完毕。
  6. 检查遍历情况： 最后，检查模式字符串是否遍历完毕，若是则返回 true，否则返回 false。

数组中的 k-diff 数对

  1. 在函数内部创建了两个哈希表 visited 和 res，分别用于存储已访问过的数和结果数对，解决了k=0的情况。
  2. 遍历数组 nums，对于每个元素 nums[i]，分别检查是否存在差值为 k 的数对。具体步骤如下：
  • 计算目标值 target = nums[i] - k，在 visited 哈希表中查找是否存在该值，如果存在，则将nums[i] - k存入 res 哈希表中。
  • 计算目标值 target = nums[i] + k，在 visited 哈希表中查找是否存在该值，如果存在，则将nums[i]该数对存入 res 哈希表中。
  • 将当前元素 nums[i] 存入 visited 哈希表中，表示已访问过。
  1. 最后返回 res 哈希表中元素的数量，即满足条件的数对数量。

同构字符串

思路同单词规律。维护两张哈希表，第一张哈希表 s2t 以 s 中字符为键，映射至 t 的字符为值，第二张哈希表 t2s 以 t 中字符为键，映射至 s 的字符为值。从左至右遍历两个字符串的字符，不断更新两张哈希表，如果出现冲突（即当前下标 index 对应的字符 s[index] 已经存在映射且不为 t[index] 或当前下标 index 对应的字符t[index] 已经存在映射且不为 s[index]）时说明两个字符串无法构成同构，返回 false。

随机链表的复制

法一：通过递归深度复制给定的随机链表，利用哈希表缓存已经复制过的节点，避免重复复制。具体步骤包括：

  • 如果输入的头节点为空，直接返回空指针。
  • 检查缓存中是否已经存在当前头节点的复制，如果存在则返回复制节点，否则进行复制并缓存。
  • 递归复制下一个节点和随机节点，并将复制后的节点链接到当前复制的节点上。

法二：

  1. 遍历原始链表，对每个节点进行复制，并将复制的节点插入到原始节点的后面。
  2. 再次遍历链表，为每个复制节点设置随机指针。由于复制节点位于原始节点的后面，因此可以通过原始节点的随机指针找到对应的复制节点的随机指针位置。
  3. 最后，将链表拆分为原始链表和复制链表，即将奇数位置的节点连接为原始链表，偶数位置的节点连接为复制链表。

2. 哈希表与索引

两数之和

  1. 初始化一个哈希表，用于存储每个数字及其对应的索引。
  2. 遍历整数数组。
  3. 对于数组中的每个数，计算与目标值的差值，并在哈希表中查找是否存在这个差值。
  4. 如果存在，说明找到了两个数使得它们的和等于目标值，返回这两个数的索引。
  5. 如果不存在，则将当前数和它的索引加入哈希表中以备后续查找。
  6. 如果遍历完整个数组仍然没有找到符合条件的两个数，则返回 NULL。

两个列表的最小索引总和

  1. 创建哈希表： 使用 struct hashTable 结构体和 UT_hash_handle 宏定义了一个哈希表。该哈希表用于存储列表1中的餐厅名称及其索引值。
  2. 遍历列表1： 使用 HASH_ADD_KEYPTR 宏将列表1中的餐厅名称和对应的索引值添加到哈希表中。
  3. 遍历列表2： 遍历列表2，对于每个餐厅名称，在哈希表中查找是否存在相同的餐厅名称。若存在，则计算列表1中的索引值和列表2中的索引值的和，并比较这些和值以确定最小索引和。
  4. 更新返回结果： 如果找到新的最小索引和，则清空之前的返回结果，将当前餐厅名称添加到返回结果中；如果找到相同的最小索引和，则将当前餐厅名称添加到返回结果中。
  5. 释放内存： 在完成所有操作后，释放哈希表中的内存。

存在重复元素II

  1. 使用哈希表来存储数组中的元素值以及其对应的索引。
  2. 遍历数组，对于每个元素：
  • 检查当前元素是否已经存在于哈希表中，如果存在，则说明已经找到了两个相同值的元素，计算它们的索引差是否不超过 k，若满足条件则返回 true，否则继续。
  • 否则，将当前元素的值和索引加入(更新)到哈希表中。
  1. 若遍历完整个数组后仍未找到满足条件的元素，则返回 false。

3. 哈希表与统计

最长和谐子序列

  1. 遍历输入的整数数组 nums，对于每个元素：
  • 使用 HASH_FIND_INT 函数在哈希表中查找该元素是否已经存在。
  • 如果存在，则将对应元素的计数值加一；如果不存在，则新建一个 hashTable 结构体表示该元素，并将其插入哈希表中。
  1. 遍历哈希表中的每个元素，对于每个元素：
  • 计算当前元素的下一个可能值 nextVal，即当前元素的值加 1。
  • 使用 HASH_FIND_INT 函数查找哈希表中是否存在该下一个可能值。
  • 如果存在，则更新最长和谐子序列的长度 max，取当前元素和下一个元素的计数值之和与 max 的较大值。

两个数组的交集II

  1. 首先，遍历 nums1 数组，将其中的元素及其出现次数存储在哈希表 set 中。
  2. 然后，遍历 nums2 数组，同时在 set 中查找元素是否存在，若存在，则将元素及其出现次数存储在哈希表 intersection 中，并在 set1 中将对应元素的出现次数减一。
  3. 最后，遍历 intersection 哈希表，根据其中元素的出现次数构建结果数组，返回结果。

砖墙

等价转换：求「垂线穿过的砖块边缘数量的最大值」，用砖墙的高度减去该最大值即为答案。

在系统中查找重复文件

通过字符串操作获取目录路径、文件名和文件内容。使用哈希表来寻找重复文件，哈希表中的键（key）是文件内容，值（value）是存储路径和文件名的列表。先遍历每一个文件，并把它加入哈希表中。在这之后，遍历哈希表，如果一个键对应的值列表的长度大于 1，说明找到了重复文件，可以把这个列表加入到答案中。

四数相加II

看到形如：A+B....+N=0的式子，要转换为(A+...T)=-((T+1)...+N)再计算，这个T的分割点一般是一半，特殊情况下需要自行判断。定T是解题的关键。

4. 哈希表与前缀和

和为K的子数组

使用前缀和的方法可以解决这个问题，因为我们需要找到和为k的连续子数组的个数。通过计算前缀和，我们可以将问题转化为求解两个前缀和之差等于k的情况。假设数组的前缀和数组为prefixSum，其中prefixSum[i]表示从数组起始位置到第i个位置的元素之和。那么对于任意的两个下标i和j（i < j），如果prefixSum[j] - prefixSum[i] = k，即从第i个位置到第j个位置的元素之和等于k，那么说明从第i+1个位置到第j个位置的连续子数组的和为k。通过遍历数组，计算每个位置的前缀和，并使用一个哈希表来存储每个前缀和出现的次数。在遍历的过程中，我们检查是否存在prefixSum[j] - k的前缀和，如果存在，说明从某个位置到当前位置的连续子数组的和为k，我们将对应的次数累加到结果中。

连续的子数组和

同余定理：如果两个整数m、n满足n-m能被k整除，那么n和m对k同余。即当 prefixSums[q]−prefixSums[p]为 k 的倍数时，prefixSums[p] 和 prefixSums[q] 除以 k 的余数相同。只需要计算每个下标对应的前缀和除以 k 的余数即可，使用哈希表存储每个余数第一次出现的下标。

连续数组

由于「0 和 1 的数量相同」等价于「1 的数量减去 0 的数量等于 0」，我们可以将数组中的 0 视作 −1，则原问题转换成「求最长的连续子数组，其元素和为 0」。遍历数组 nums，当遇到元素 1 时将 counter的值加 1，当遇到元素 0 时将 counter 的值减 1，遍历过程中使用哈希表存储每个前缀和第一次出现的下标。
