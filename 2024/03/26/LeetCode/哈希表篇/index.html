设计哈希集合

实现了一个基于链表的哈希集合数据结构，并提供了添加、删除、查询和释放内存的功能。注：链表中使用了虚拟头节点。

设计哈希映射

「设计哈希映射」与「设计哈希集合」解法接近，唯一的区别在于我们存储的不是 key 本身，而是 (key,value)(key,value) 对。除此之外，代码基本是类似的。

存在重复元素

法一：在对数字从小到大排序之后，数组的重复元素一定出现在相邻位置中。因此，我们可以扫描已排序的数组，每次判断相邻的两个元素是否相等，如果相等则说明存在重复的元素。注：冒泡排序会超时。

qsort 是 C 标准库中的一个函数，用于对数组进行快速排序。它的声明如下：

参数解释： - base：指向要排序的数组的指针，即数组的首元素地址。 - nmemb：数组中元素的个数。 - size：每个元素的大小，以字节为单位。 - compar：指向一个用来比较两个元素的函数的指针。这个函数需要接受两个指向常量的指针，分别指向待比较的元素，并返回一个整数值，表示它们的大小关系。如果第一个参数小于第二个参数，函数应该返回一个负数；如果第一个参数大于第二个参数，函数应该返回一个正数；如果两个参数相等，函数应该返回 0。

qsort 函数使用快速排序算法对数组进行排序，其时间复杂度为 O(n*log(n))。排序完成后，数组中的元素将按照给定的比较函数的规则进行排序。

法二：对于数组中每个元素，我们将它插入到哈希表中。如果插入一个元素时发现该元素已经存在于哈希表中，则说明存在重复的元素。

附：

  1. HASH_ADD_INT(table, field, ptr)
  • table: 哈希表的头指针，即存储数据结构的指针。
  • field: 数据结构中作为哈希键的字段名。
  • ptr: 指向要插入的数据结构的指针。
  1. HASH_DEL(table, ptr)
  • table: 哈希表的头指针。
  • ptr: 指向要删除的数据结构的指针。
  1. HASH_FIND_INT(table, findptr, outptr)
  • table: 哈希表的头指针。
  • findptr: 指向要查找的键的值的指针。
  • outptr: 用于接收查找结果的指针。
  1. HASH_ITER(hh, table, el, tmp)
  • hh: 表示哈希表的哈希句柄字段的名称。在大多数情况下，这个字段是用来将元素链接到哈希表的。
  • table: 哈希表的头指针，即指向哈希表中第一个元素的指针。
  • el: 在每次迭代中，用于接收当前迭代元素的指针。这是一个临时指针，用于在迭代过程中遍历哈希表中的每个元素。
  • tmp: 一个临时指针，用于保存下一个要迭代的元素的指针。在循环的每次迭代中，el指向当前元素，而tmp指向下一个元素。

通过这些宏，我们可以方便地操作哈希表，实现插入、删除、查找和遍历等功能。

两个数组的交集

遍历数组 nums1，对于其中的每个元素，遍历数组 nums2 判断该元素是否在数组 nums2 中。使用了哈希表降低时间复杂度，以空间换时间。

最长连续序列

法一：通过排序数组，并使用线性扫描的方式，以时间复杂度 O(n log n) 找到最长连续序列的长度。

法二：

  1. 首先，将数组中的所有元素存入一个哈希表中，以便快速检查某个元素是否存在。
  2. 然后，对数组中的每个元素进行遍历，对于每个元素，检查其是否是一个序列的起点。如果一个数字 x 是序列的起点，则从 x+1 开始依次检查是否存在于哈希表中，直到找到序列的结束。在此过程中，记录当前连续序列的长度。
  3. 遍历数组中的每个元素并执行上述操作，最终得到数字连续的最长序列的长度。
