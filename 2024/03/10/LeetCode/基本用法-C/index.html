基本用法-C

1. 字符串

1.1 strcpy()

strcpy() 函数用于把 src 所指向的字符串复制到 dest。需要注意的是如果目标数组 dest 不够大，而源字符串的长度又太长，可能会造成缓冲溢出的情况。其函数声明如下：

  • dest ：指向用于存储复制内容的目标数组。
  • src ： 要复制的字符串。

返回值：

该函数返回一个指向最终的目标字符串 dest 的指针。

1.2 strncpy()

strncpy() 函数用于把 src 所指向的字符串复制到 dest，最多复制 n 个字符。当 src 的长度小于 n 时，dest 的剩余部分将用空字节填充。其函数声明如下：

  • dest：指向用于存储复制内容的目标数组。
  • src：要复制的字符串。
  • n：要从源中复制的字符数。

返回值：

该函数返回一个指向最终的目标字符串 dest 的指针。

1.3 strcmp()

strcmp() 用于把 str1 所指向的字符串和 str2 所指向的字符串进行比较。其函数声明如下：

  • str1：要进行比较的第一个字符串。
  • str2：要进行比较的第二个字符串。

返回值：

  • 如果返回值小于 0，则表示 str1 小于 str2。
  • 如果返回值大于 0，则表示 str1 大于 str2。
  • 如果返回值等于 0，则表示 str1 等于 str2。

1.4 strdup()

strdup() 会先用 maolloc（）配置与参数 s 字符串相同的空间大小，然后将参数 s 字符串的内容复制到该内存地址，然后把该地址返回。函数的声明如下：

  • s: 待复制的字符串。

返回值：

返回一字符串指针，该指针指向复制后的新字符串地址。若返回NULL 表示内存不足。

1.5 strtok()

strtok()用于分解字符串 str 为一组字符串，delim 为分隔符。函数的声明如下：

  • str ：要被分解成一组小字符串的字符串。
  • delim ：包含分隔符的 C 字符串。

返回值：

该函数返回被分解的第一个子字符串，如果没有可检索的字符串，则返回一个空指针。

1.6 strcat()

strcat() 用于把 src 所指向的字符串追加到 dest 所指向的字符串的结尾。函数的声明如下：

  • dest ： 指向目标数组，该数组包含了一个 C 字符串，且足够容纳追加后的字符串。
  • src ： 指向要追加的字符串，该字符串不会覆盖目标字符串。

返回值：

该函数返回一个指向最终的目标字符串 dest 的指针。

1.7 strstr()

strstr() 在字符串 haystack 中查找第一次出现字符串 needle 的位置，不包含终止符 '\0'。函数的声明如下：

  • haystack ： 要被检索的 C 字符串。
  • needle ： 在 haystack 字符串内要搜索的小字符串。

返回值：

该函数返回在 haystack 中第一次出现 needle 字符串的位置，如果未找到则返回 null。

2. 哈希表

uthash 是一个 C 语言的哈希表函数库，支持哈希表的各种操作，包括添加，删除，查找，排序等。

2.1 数据结构

uthash 的哈希表是由多个结构体组成的双向链表实现的，一个结构体就是一个键值对，例如：将id作为key，将name作为value。

  • 键（key）：名称和数据类型没有限制；
  • 值（value）：名称和数据类型没有限制；
  • UT_hash_handle 字段必须存在于结构体中，hh 是内部使用的 hash 处理句柄。它用于内部记录，使哈希表正常工作，它不需要初始化。它可以命名为任何名称，但是我们一般都命名为hh。

2.2 声明

定义一个结构体类型的空指针，即声明了一个哈希表。此时的哈希表示空的，我们需要向表中添加数据元素，在整个生命周期中，这个指针都指向链表的头结点。

2.3 添加

  • HASH_ADD_PTR(head, keyfield_name, item_ptr)：添加新节点，键值是指针
  • HASH_ADD(hh_name, head, keyfield_name, key_len, item_ptr)：添加新节点，不关心类型

2.4 查询

  • HASH_FIND_INT(head, key_ptr, item_ptr)： 查找指定的节点，键值是整数
  • HASH_FIND_STR(head, key_ptr, item_ptr)： 查找指定的节点，键值是字符串

2.5 替换

  • HASH_REPLACE_INT(head, keyfiled_name, item_ptr,replaced_item_ptr)
  • HASH_REPLACE(hh_name, head, keyfield_name, key_len, item_ptr, replaced_item_ptr)

HASH_REPLACE宏等效于HASH_ADD宏，HASH_REPLACE会尝试查找和删除项目。如果找到并删除了一个项目，它还将返回该项目的指针作为输出参数。

3. 内存分配

3.1 malloc()

malloc()用于 分配所需的内存空间，并返回一个指向它的指针。其函数声明如下：

  • size ：内存块的大小，以字节为单位。

返回值：

该函数返回一个指针 ，指向已分配大小的内存。如果请求失败，则返回 NULL。

3.2 calloc()

calloc()用于 分配所需的内存空间，并返回一个指向它的指针。malloc 和 calloc 之间的不同点是，malloc 不会设置内存为零，而 calloc 会设置分配的内存为零。其函数声明如下：

注意：calloc() 函数将分配的内存全部初始化为零。如果不需要初始化，可以使用 malloc() 函数代替。另外，使用 calloc() 函数时需要注意，如果分配的内存块过大，可能会导致内存不足的问题。

  • nitems ： 要被分配的元素个数。
  • size ： 元素的大小。

返回值：

该函数返回一个指针，指向已分配的内存。如果请求失败，则返回 NULL。

3.3 realloc()

realloc()用于尝试重新调整之前调用 malloc 或 calloc 所分配的 ptr 所指向的内存块的大小。其函数声明如下：

  • ptr ：指针指向一个要重新分配内存的内存块，该内存块之前是通过调用 malloc、calloc 或 realloc 进行分配内存的。如果为空指针，则会分配一个新的内存块，且函数返回一个指向它的指针。
  • size* ： 内存块的新的大小，以字节为单位。如果大小为 0，且 ptr 指向一个已存在的内存块，则 ptr 所指向的内存块会被释放，并返回一个空指针。

返回值：

该函数返回一个指针 ，指向重新分配大小的内存。如果请求失败，则返回 NULL。
