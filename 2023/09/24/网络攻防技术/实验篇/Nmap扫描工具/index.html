Nmap扫描工具

1. 简介

1.1 概述

Nmap（“ Network Mapper ”）是一个用于网络探测和安全审计的开源工具。它被设计用于快速扫描大型网络，尽管它对于单个主机也能正常工作。Nmap 以新颖的方式使用原始IP数据包来确定网络上可用的主机、这些主机提供的服务（应用程序名称和版本）、它们运行的操作系统及其版本、正在使用中的数据包过滤器/防火墙的类型，以及数十个其他特性。

1.2 功能

  • 主机发现
  • 端口扫描
  • 推断主机所用的操作系统

1.3 常见用法

  • -iL (从列表中输入)：可将文件中的一堆主机名或地址读取出来进行扫描。

  • -iR (随机选择目标)：可以指定随机生成多少个IP地址。如果为0，意味着无休止的扫描。

  • –exclude：可以排除多个不需要扫描的主机。

  • –excludfile (排除文件中的列表)：排除文件中的主机列表

2. 主机发现

Nmap 会根据当前扫描的网络来改变它的扫描方式：

  • 本地网络发送 ARP 数据包；
  • 非本地网路依次发送：A) ICMP echo 请求；B) TCP SYN 到端口 443；C) TCP ACK 到端口 80；D) ICMP timestap 请求。

2.1 -sL(列表扫描)

列表扫描是主机发现的一种退化形式，它仅列出指定网络的每个主机，而不向目标主机发送任何数据包，即不进行主机发现。默认情况下，Nmap 仍然在主机上执行反向 DNS 解析（IP→域名）以了解其名称。Nmap 还在末尾报告 IP 地址的总数。由于这个想法是简单地打印目标主机列表，因此端口扫描、操作系统检测或主机发现等更高级别功能的选项不能与此结合。

2.2 -sn(无端口扫描)

该选项告诉 Nmap 在主机发现后不要进行端口扫描，而只打印出响应主机发现探测的可用主机。默认情况下，完成的默认主机发现-sn由 ICMP 回显请求、端口 443 的 TCP SYN、端口 80 的 TCP ACK 以及ICMP时间戳请求组成。当由非特权用户执行时，仅将 SYN 数据包发送（使用调用connect）到目标上的端口 80 和 443。当特权用户尝试扫描本地以太网上的目标时，除非 --send-ip指定，否则将使用 ARP 请求。该-sn选项可以与任何发现探测类型（-P*选项）以获得更大的灵活性。如果使用任何这些探测类型和端口号选项，则默认探测将被覆盖。当运行 Nmap 的源主机和目标网络之间存在严格的防火墙时，建议使用这些高级技术。否则，当防火墙丢弃探测或其响应时，主机可能会丢失。只利用ping扫描进行主机发现，不扫描目标主机的端口。在 Nmap 的早期版本中，-sn被称为 -sP。·

2.3 -Pn(无 ping)

该选项完全跳过主机发现阶段。通常情况下，Nmap 使用此阶段来确定活动机器，以便进行更大规模的扫描，并衡量网络速度。与列表扫描一样，会跳过正确的主机发现，但Nmap不会停止并打印目标列表，而是继续执行请求的功能，就好像每个目标 IP 都处于活动状态一样。默认情况下，Nmap 只对发现运行的主机执行重型探测，如端口扫描、版本检测或操作系统检测。对于本地以太网上的机器，仍然会执行 ARP 扫描（除非 指定--disable-arp-ping或--send-ip），因为Nmap 需要 MAC 地址来进一步扫描目标主机。在 Nmap 的早期版本中，-Pn是 -P0 和 -PN。

2.4 -PS (TCP SYN Ping)

该选项会发送一个设置了SYN标志的空 TCP 数据包。默认目标端口为 80。SYN 标志向远程系统表明，您正试图建立连接。通常情况下，目标端口会被关闭，并回发一个 RST（重置）数据包。如果端口碰巧是开放的，目标系统将采取 TCP 三方握手的第二步，回应一个 SYN/ACK TCP 数据包。然后，运行 Nmap 的机器会以 RST 回应，而不是发送 ACK 数据包（ACK 数据包将完成三方握手并建立完整连接）来破坏新生连接。Nmap 不关心端口是打开还是关闭。前面讨论的 RST 或 SYN/ACK 响应告诉 Nmap 主机可用且有响应。注意：-PS和端口列表间无空格。

2.5 -PA (TCP ACK Ping)

TCP ACK ping 与刚才讨论的 SYN ping 非常相似。两者的区别在于设置的是 TCP ACK 标志，而不是 SYN 标志。同时提供 SYN 和 ACK ping 探测的原因是为了最大限度地绕过防火墙。该-PA选项使用与 SYN 探测 (80) 相同的默认端口，并且还可以采用相同格式的目标端口列表。这种 ACK 数据包声称通过已建立的 TCP 连接确认数据，但实际上并不存在这样的连接。因此，远程主机应始终响应一个 RST 数据包，在此过程中暴露自己的存在。

2.6 -PU (UDP Ping)

该方法会向指定端口发送 UDP 数据包。对于大多数端口，数据包都是空的，但有些端口会使用特定协议的有效载荷，这样更有可能得到响应。端口列表的格式与前面讨论过的 -PS 和 -PA 选项相同。如果没有指定端口，默认值为 40125。默认情况下使用的是一个非常不常用的端口，因为对于这种特殊的扫描类型来说，向开放端口发送数据通常是不可取的。一旦命中中目标计算机上的一个关闭端口，UDP 探测器应返回一个 ICMP 端口不可达数据包。这就向 Nmap 表明机器是正常运行的。许多其他类型的 ICMP 错误，如主机/网络不可到达或超过 TTL，都表明主机已停机或不可到达。无响应也可以这样解释。如果到达的是开放端口，大多数服务会直接忽略空数据包，不返回任何响应。这就是默认探测端口为 40125 的原因，该端口被使用的可能性很小。字符生成器 chargen协议等少数服务会响应空 UDP 数据包，从而向 Nmap 透露机器可用。

2.7 -PY (SCTP INIT Ping)

该选项发送包含最小 INIT 块的 SCTP 数据包。INIT 块向远程系统表明，您正试图建立关联。通常情况下，目标端口关闭，并将发送一个 ABORT 块。如果端口碰巧是开放的，目标系统将采取 SCTP 四次握手的第二步，用一个 INIT-ACK chunk 作为回应。如果运行 Nmap 的机器有一个功能正常的SCTP协议栈，那么它就会通过响应一个ABORT数据块而不是发送一个 COOKIE-ECHO 数据块（这将是四路握手的下一步）来破坏连接。ABORT 包由运行Nmap的机器内核发送，以响应意外的 INIT-ACK，而不是由 Nmap 本身发送。前面讨论过的 ABORT 或 INIT-ACK 响应都会告诉 Nmap，主机是可用的并有响应。

2.8 -PE; -PP; -PM (ICMP Ping Types)

Nmap 向目标IP地址发送ICMP类型 8（回显请求）数据包，期望从可用主机返回类型 0（回显应答）。 不幸的是，对于网络浏览器来说，许多主机和防火墙现在会阻止这些数据包，而不是按照RFC 1122 的要求进行响应。 因此，针对 Internet 上的未知目标，仅 ICMP 扫描很少足够可靠。但对于监控内部网络的系统管理员来说，它们可能是一种实用且有效的方法。使用该-PE选项启用此回显请求行为。ICMP 标准（RFC 792 和 RFC 950 ）还将时间戳请求、信息请求和地址掩码请求数据包分别指定为代码 13、15 和 17。时间戳和地址掩码查询可以分别使用 -PP和-PM选项发送。

2.9 -PO (IP 协议 Ping)

IP 协议 ping 是较新的主机发现选项之一，它发送的 IP 数据包在其 IP 头中设置了指定的协议号。协议列表的格式与前面讨论的 TCP、UDP 和 SCTP 主机发现选项中的端口列表相同。此主机发现方法查找使用与探针相同的协议的响应，或者查找表示目标主机不支持给定协议的 ICMP 协议不可达消息。无论哪种类型的响应都表明目标主机存活。

2.10 -PR(ARP Ping)

当启动Namp主机发现时候，Nmap会对目标地址参数进行检查，如果与自身IP地址匹配到同一个子网内，Nmap会对该类目标采用ARP协议进行探测。即使命令参数规定的是其他探测手段，也会先使用ARP进行探测。这种方式效率高，速度快，但仅限于同一子网广播域中。ARP（Address Resolution Protocol，地址解析协议）是用来将IP地址解析为MAC地址的协议。主机或三层网络设备上会维护一张ARP表，用于存储IP地址和MAC地址的映射关系，一般ARP表项包括动态ARP表项和静态ARP表项。

2.11 –disable-arp-ping (No ARP or ND Ping)

Nmap 通常会发现本地连接的以太网主机的 ARP 或 IPv6 邻居发现 (ND)，即使使用了 -Pn 或 -PE 等其他主机发现选项。要禁用这种隐式行为，请使用--disable-arp-ping选项。

2.12 –discovery-ignore-rst

在某些情况下，防火墙可能会欺骗 TCP 重置 (RST) 回复，以响应对未占用或不允许的地址的探测。由于 Nmap 通常会将 RST 回复视为目标已启动的证明，这可能导致浪费扫描不存在的目标的时间。使用 --discovery-ignore-rst 会阻止 Nmap 在发现主机时考虑这些回复。您可能需要选择额外的主机发现选项，以确保在这种情况下不会错过目标。

2.13 –traceroute(跟踪主机路径)

Traceroute 的工作原理是发送 TTL（生存时间）较低的数据包，试图从扫描器和目标主机之间的中间跳诱发ICMP时间超限信息。标准的 traceroute 实现以 1 的 TTL 开始，并递增 TTL 直到到达目标主机。Nmap的traceroute从高TTL开始，然后递减TTL，直到TTL为零。这样做可以让 Nmap 采用巧妙的缓存算法，加快对多台主机的跟踪。根据网络条件，Nmap 平均每台主机少发送 5-10 个数据包。如果扫描的是单个子网（如 192.168.0.0/24），Nmap 可能只需向大多数主机发送两个数据包。
