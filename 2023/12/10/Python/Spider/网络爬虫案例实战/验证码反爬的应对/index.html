验证码反爬的应对

0. 引言

有些网站为了避免被过度访问，会设置验证码反爬机制，如果访问次数过多就要求用户输入验证码，甚至一开始访问时就要求输入验证码。验证码类型很多，如：图像验证码、计算题验证码、滑块验证码、滑动拼图验证码、点选验证码。

0.1 超级鹰注册

超级鹰：验证码识别平台

0.2 超级鹰接口

Python语言Demo下载：下载得到一个压缩包，解压后得到的chaojiying.py就是示例代码，然后将该文件拷贝至代码文件目录下。

0.3 软件ID

生成软件ID，以供后续使用：用户中心→软件ID→生成一个软件ID

0.4 调用示例

验证码类型编号：不同验证码类型，选择不同编号值。

\在python中有特殊含义，如\n表示换行。在书写绝对路径时需要使用\\，或者在字符串前加一个字母r，或者用/。

1.2 中文验证码

2. 计算题验证码

它与普通图像验证码的区别在于增加了数学运算，需要将验证码中数学计算题的结果填到文本框中，再点击验证。计算题验证码的处理难点在于运算符号的识别，但超级鹰已经提供了用于识别此类验证码的接口。

代码修改：PostPic()的验证码类型参数修改为6001。

3. 滑块验证码

这种验证码的验证机制比较简单，将滑块拖动到滑轨的最右端即可完成验证。

验证思路：调用Selenium库，打开网页，定位滑块，模拟鼠标操作，移动滑块至滑轨最右端。（开发者工具查看滑块滑轨尺寸）

注：如果网站采取了webdriver反爬，处理起来会比较困难。解决方法：如果是登录阶段需要进行滑块验证，则可以在代码中使用time.sleep()等待一段时间，在这段时间内使用手动登录，之后再用Selenium库继续爬取。

4. 滑动拼图验证码

  • 初级版：网页源码中包含缺口位置信息
  • 高级版：网页源码中不包含缺口位置信息

4.1 初级版

初级版滑动拼图验证码是在普通滑块验证码的基础上增加了随机的滑动距离，用户需要根据拼图的缺口位置来决定滑块的滑动距离。

模拟缓慢滑动：将滑动距离分段，让滑块分多次滑动，每次滑动后等待一定时间。

4.2 高级版

使用PIL库pip install pillow对比无缺口和有缺口的图像，从而计算出滑块需要滑动的距离。

ImageChops.difference(image_a,image_b).getbbox：返回坐标值元组。前两个元素为缺口左上角的x坐标和y坐标，后两个元素为缺口右下角的x坐标和y坐标。

解决方案：Python3 PIL库问题：ImageChops.difference返回None-CSDN博客

5. 点选验证码

通过传统的图像识别方法进行识别比较麻烦，但超级鹰提供了相应的接口，类型代码为9004。返回格式：282,54|489,59|513,44|342,157。选择图片保存时，需涵盖上方的图片和下方的文字说明。
